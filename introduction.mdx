## What is LLVM?

LLVM is a toolkit for the construction of highly optimized compilers, optimizers, and runtime environments. At its core, LLVM defines a language-independent intermediate representation (IR) — a strongly typed, low-level instruction set that serves as the common currency between frontends, middle-end optimization passes, and target-specific backends.

The project provides:

- **LLVM IR** — A well-specified, SSA-form intermediate representation in both human-readable (`.ll`) and binary bitcode (`.bc`) formats.
- **Pass system** — A pipeline of analysis and transformation passes that operate on IR. Passes can be composed and ordered to perform optimizations such as dead code elimination, inlining, loop transformations, and vectorization.
- **Backends** — Target-specific code generators that lower LLVM IR to machine code for a given architecture. The code generator is divided into instruction selection, instruction scheduling, and register allocation phases.
- **Tools** — A set of standalone utilities including `llc` (the static compiler), `opt` (the IR optimizer), `llvm-as`/`llvm-dis` (assembler/disassembler), `lli` (the interpreter and JIT), and `llvm-ar` (the archiver).

<Note>
All LLVM sub-projects are developed together in a single **monorepo** at [github.com/llvm/llvm-project](https://github.com/llvm/llvm-project). The core LLVM libraries live under `llvm/`, while each sub-project (Clang, LLD, LLDB, etc.) occupies its own top-level directory.
</Note>

The monorepo also includes:

- **libc++** — An implementation of the C++ standard library.
- **libc++abi** — The low-level C++ ABI library.
- **compiler-rt** — Runtime support libraries for sanitizers (AddressSanitizer, MemorySanitizer, etc.) and profiling.
- **polly** — A high-level loop optimizer using polyhedral model techniques.
- **clang-tools-extra** — Additional Clang-based tools including `clang-tidy` and `clangd`.

## Key use cases

**Building compilers**
LLVM provides all the components needed to construct a complete compiler: a well-defined IR to target, a rich set of analyses and optimizations, and target backends for every major architecture. New language frontends emit LLVM IR and immediately gain access to the full LLVM optimization and code generation pipeline.

**Writing passes**
The LLVM pass infrastructure lets you write modular analysis and transformation passes in C++. Passes can inspect or rewrite IR, query other analyses (e.g., dominator trees, alias analysis, loop information), and be composed into custom optimization pipelines using the new pass manager.

**JIT compilation**
LLVM's ORC (On-Request Compilation) JIT APIs allow you to compile and execute LLVM IR at runtime. The `lli` tool demonstrates this; the `BuildingAJIT` tutorial series shows how to construct a custom JIT suited to your use case.

## Supported targets

LLVM includes production-quality backends for the following architectures:

| Architecture | Notes |
|---|---|
| x86 / x86-64 | Primary development and CI target |
| ARM / AArch64 | Used on Linux, macOS (Apple Silicon), iOS |
| RISC-V | RV32 and RV64, with vector extension support |
| AMDGPU | ROCm GPU compute and graphics |
| NVPTX | NVIDIA GPU code generation via PTX |
| PowerPC | Linux and AIX targets |
| SystemZ | IBM Z mainframe |
| MIPS | 32-bit and 64-bit variants |
| WebAssembly | wasm32 and wasm64 |
| LoongArch | LoongArch 32 and 64-bit |
| Sparc | 32-bit and V9 |
| Hexagon | Qualcomm DSP target |

LLVM is known to work on Linux (x86, amd64, ARM, AArch64, LoongArch, MIPS, PowerPC, RISC-V, SystemZ), macOS (x86, arm64), FreeBSD, NetBSD, OpenBSD, Solaris, and Windows (x86, x86-64, ARM64).
